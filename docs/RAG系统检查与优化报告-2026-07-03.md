## 🐛 已修复的 Bug（2 个关键 Bug）

### Bug 1: `searchFAQCandidates` 置信度阈值过高，过滤掉所有结果

**症状**: `/api/eval/rag` 对任何查询都返回 0 条候选

**根因**:
1. `searchByFAQCacheAsync` 返回的 `r.rerankScore` 值极小（~0.00035）
2. 第 346 行 `let confidence = r.rerankScore || r.score`：`||` 运算符把极小值当作 truthy，导致置信度算出来接近 0
3. 第 350 行 `if (confidence < 0.25) continue`：阈值 0.25 过高，所有结果被过滤

**修复**（`index.js` 第 345-351 行）:
```javascript
// 修复前
let confidence = r.rerankScore || r.score;
if (confidence < 0.25) continue;

// 修复后
let confidence = r.score || 0;  // 优先用余弦相似度
if (r.rerankScore !== undefined && r.rerankScore > confidence + 0.2) {
  confidence = r.rerankScore;  // 仅当 Rerank 显著更高时才用
}
if (confidence < 0.05) continue;  // 阈值降至 0.05
```

---

### Bug 2: `searchByFAQCacheAsync` 返回的 `score` 是 RRF 分数，不是余弦相似度

**症状**: 上游代码用 `r.score` 作为置信度，但实际拿到的是 RRF 融合分数（~0.016），导致置信度计算错误

**根因**: `vector-store.js` 第 536 行 `score: r.score`：`r.score` 是 RRF 融合分数（值域 ~0-1/N），不是余弦相似度（值域 0-1）

**修复**（`vector-store.js` 第 534-541 行）:
```javascript
// 修复前
score: r.score,  // RRF 分数，极小

// 修复后
score: r.originalScore || r.score,  // 保留余弦相似度
rrfScore: r.score,                  // RRF 分数另存
```
