# 模型性能监控功能说明

## 📊 功能概述

模型性能监控功能用于实时监控Ollama模型的性能指标，包括：
- 响应时间（平均、最小、最大、最后）
- 成功率/失败率
- 请求吞吐量（每分钟请求数）
- 健康状态
- 自动告警（响应时间过高、成功率过低、请求频率过高）

---

## 🔧 已实现的API接口

### 1. 获取模型状态（包含性能数据）
```
GET /api/admin/models/status
Headers: Authorization: Bearer <token>

响应示例：
{
  "success": true,
  "currentModels": {
    "embedding": "bge-m3:latest",
    "llm": "qwen2.5:14b",
    "reranker": "bge-reranker-v2-m3:latest"
  },
  "health": { ... },
  "performance": {
    "embedding": {
      "totalRequests": 10,
      "successRate": "100.00%",
      "avgResponseTime": "375ms",
      ...
    },
    ...
  }
}
```

### 2. 获取详细性能报告
```
GET /api/admin/models/performance
Headers: Authorization: Bearer <token>

# 或指定模型类型
GET /api/admin/models/performance?type=embedding
```

### 3. 重置性能统计
```
POST /api/admin/models/performance/reset
Headers: Authorization: Bearer <token>
Body: { "type": "embedding" }  # 可选，不指定则重置所有
```

---

## 📈 性能数据说明

| 指标 | 说明 | 单位 |
|------|------|------|
| **totalRequests** | 总请求数 | 次 |
| **successRequests** | 成功请求数 | 次 |
| **failedRequests** | 失败请求数 | 次 |
| **successRate** | 成功率 | % |
| **avgResponseTime** | 平均响应时间 | ms |
| **minResponseTime** | 最小响应时间 | ms |
| **maxResponseTime** | 最大响应时间 | ms |
| **lastResponseTime** | 最后一次响应时间 | ms |
| **requestsPerMinute** | 每分钟请求数 | 次/分钟 |

---

## ⚙️ 自动保存

性能数据每5分钟自动保存到：
```
server/data/model-performance.json
```

后端重启后会自动加载历史性能数据。

---

## 🚨 告警阈值

| 指标 | 阈值 | 说明 |
|------|------|------|
| 平均响应时间 | > 5000ms | 响应时间过高 |
| 成功率 | < 80% | 成功率过低 |
| 请求频率 | > 100次/分钟 | 请求频率过高 |

达到阈值会在后端日志中输出告警信息。
