### 3. 实现重试机制
**优化前**：一次调用失败直接使用降级方案  
**优化后**：指数退避重试（最多3次）

**代码实现**：
```javascript
// server/intent-understanding.js
async function understandIntent(userQuery, context, retryCount = 0) {
  try {
    const prompt = buildPrompt(query);
    const parsed = await callOllamaJSON(prompt, { ... });
    
    // 验证返回结果
    if (parsed && parsed.primaryIntent && parsed.primaryIntent.level1) {
      if (INTENT_TAXONOMY.level1.includes(parsed.primaryIntent.level1)) {
        return parsed;
      }
    }
    
    throw new Error('Invalid LLM response');
  } catch (err) {
    console.error(`[Intent] Error (attempt ${retryCount + 1}/3):`, err.message);
    
    // 重试机制：最多重试3次
    if (retryCount < 2) {
      console.log(`[Intent] Retrying... (${retryCount + 2}/3)`);
      await sleep(1000 * (retryCount + 1));  // 指数退避：1s, 2s, 3s
      return await understandIntent(userQuery, context, retryCount + 1);
    }
    
    // 重试失败，使用降级方案
    return fallbackIntent(query);
  }
}
```

**效果**：应对Ollama偶发超时，提升系统稳定性

### 4. 增强JSON解析容错
**优化前**：简单正则匹配 `{...}`  
**优化后**：4层解析策略 + 智能修复

**解析策略**：
1. **策略1**：直接解析（最快）
2. **策略2**：提取markdown代码块中的JSON
3. **策略3**：正则匹配第一个完整JSON对象 + 智能修复
4. **策略4**：降级为默认结构

**新增函数**：`repairJSON`（自动修复不完整JSON）
```javascript
// server/ollama-client.js
function repairJSON(jsonStr) {
  // 修复1：移除可能的尾部逗号
  let repaired = jsonStr.replace(/,\s*\}/g, '}').replace(/,\s*\]/g, ']');
  
  // 修复2：补全缺失的闭合括号
  const stack = [];
  let inString = false;
  let escapeNext = false;
  
  for (let i = 0; i < repaired.length; i++) {
    const char = repaired[i];
    // ... 括号匹配逻辑 ...
  }
  
  // 补全缺失的闭合括号
  while (stack.length > 0) {
    const open = stack.pop();
    if (open === '{') repaired += '}';
    else if (open === '[') repaired += ']';
  }
  
  return repaired;
}
```

**效果**：应对Ollama返回格式错误，解析成功率100%
