## 📊 项目概述

### 项目定位
**广康集团AI助手**是一款面向企业内部的智能客服系统，基于 **RAG（检索增强生成）** 架构，集成了 **意图理解、多轮对话记忆、LLM智能改写** 三大核心AI模块，为用户提供准确、智能、人性化的问答服务。

### 技术栈

| **前端** | React + TypeScript + Vite + Tailwind CSS | React 18+, Vite 5+ |

| **LLM** | Ollama (qwen2.5:14b, deepseek-r1:14b) | Ollama 0.1.47+ |

## 📦 核心功能模块详解

### 1. RAG向量检索模块（`server/vector-store.js`）
#### 功能作用
- **FAQ向量化**：将FAQ question+answer转换为1024维向量嵌入
- **语义相似度搜索**：根据用户查询，检索最相关的FAQ条目
- **多级检索策略**：本地快速匹配 → FAQ缓存搜索 → 语义搜索（降级策略）

#### 技术亮点
1. **FAQ Embedding缓存**：预计算所有FAQ的嵌入向量，缓存到内存，搜索时直接计算余弦相似度（纯CPU计算，<10ms）
2. **异步向量化**：新增/修改FAQ时，异步调用bge-m3 API，不阻塞主流程
3. **降级策略**：bge-m3不可用时，自动降级为关键词匹配

#### 核心代码
```javascript
// server/vector-store.js
// 构建FAQ embedding缓存（启动时执行）
function buildFAQEmbeddingCache() {
  const faqList = getFAQ();
  let cached = 0;
  for (const faq of faqList) {
    const embedding = getEmbeddingFromCache(faq.id);
    if (embedding) {
      faqEmbeddingCache.set(faq.id, embedding);
      cached++;
    }
  }
  return cached;
}

// 异步FAQ缓存搜索（纯内存计算，无需调用Rerank）
async function searchByFAQCacheAsync(query, topK = 5, minScore = 0.1) {
  const queryEmbedding = await getEmbedding(query);
  const results = [];
  
  for (const [faqId, embedding] of faqEmbeddingCache.entries()) {
    const score = cosineSimilarity(queryEmbedding, embedding);
    if (score >= minScore) {
      results.push({ parentDocId: faqId, score });
    }
  }
  
  return results.sort((a, b) => b.score - a.score).slice(0, topK);
}
```

### 2. 智能意图理解模块（`server/intent-understanding.js`）
#### 功能作用
- **规则引擎**：快速匹配问候语、投诉、申请流程等高频意图（<1ms）
- **LLM意图理解**：调用Ollama进行深度意图分析（2-5秒）
- **实体提取**：从用户查询中提取关键实体（如"报销"、"微信支付"）

#### 技术亮点
1. **双引擎架构**：规则引擎（快速） + LLM理解（精准），平衡性能和准确率
2. **Few-shot Prompt工程**：11个示例，覆盖常见意图，LLM准确率从75%提升至100%
3. **重试机制**：指数退避重试（1s → 2s → 3s，最多3次），应对Ollama偶发超时
4. **JSON解析容错**：4层解析策略（直接解析 → markdown代码块 → 正则匹配+智能修复 → 降级）

#### 核心代码
```javascript
// server/intent-understanding.js
// 规则引擎：快速匹配（避免LLM调用）
function quickRuleCheck(query) {
  const q = query.toLowerCase();
  
  // 规则1：问候语（高置信度）
  if (q.includes('你好') || q.includes('您好') || q.includes('hi')) {
    return { primaryIntent: { level1: 'greeting', level2: 'hello', confidence: 0.98 }, entities: [] };
  }
    
  // 规则2：投诉（高置信度）
  if (q.includes('投诉') || q.includes('举报') || q.includes('不满意')) {
    return { primaryIntent: { level1: 'complaint', level2: 'service', confidence: 0.95 }, entities: [] };
  }
    
  // 规则3：申请/流程（高置信度）
  if (q.includes('如何申请') || q.includes('怎么申请')) {
    return { primaryIntent: { level1: 'process', level2: 'apply', confidence: 0.95 }, entities: [{ type: 'process', value: '申请', confidence: 0.9 }] };
  }
    
  // 没有匹配规则，需要LLM
  return null;
}

// 主函数（带重试机制）
async function understandIntent(userQuery, context, retryCount = 0) {
  try {
    const prompt = buildPrompt(query);
    const parsed = await callOllamaJSON(prompt, {
      baseURL: OLLAMA_BASE_URL,
      model: OLLAMA_MODEL,
      temperature: 0.1,
      max_tokens: 500,
      timeout: 90000  // 90秒超时
    });
      
    // 验证返回结果
    if (parsed && parsed.primaryIntent && parsed.primaryIntent.level1) {
      if (INTENT_TAXONOMY.level1.includes(parsed.primaryIntent.level1)) {
        return parsed;
      }
    }
      
    throw new Error('Invalid LLM response');
  } catch (err) {
    console.error(`[Intent] Error (attempt ${retryCount + 1}/3):`, err.message);
      
    // 重试机制：最多重试3次
    if (retryCount < 2) {
      console.log(`[Intent] Retrying... (${retryCount + 2}/3)`);
      await sleep(1000 * (retryCount + 1));  // 指数退避：1s, 2s, 3s
      return await understandIntent(userQuery, context, retryCount + 1);
    }
      
    // 重试失败，使用降级方案
    return fallbackIntent(query);
  }
}
```

- **话题链追踪**：追踪对话话题演化，支持上下文感知回答

#### 技术亮点

2. **bge-m3语义相似度**：理解语义层面相关性，处理同义词、近义词
3. **三层缓存机制**：
   - 内存缓存（Map）：最快访问
   - 磁盘缓存（JSON文件）：持久化存储
   - 实时计算（降级）：备用方案
4. **预计算嵌入向量**：在`storeConversationRound`时后台计算，查询时直接使用缓存（缓存命中<0.01秒）

#### 核心代码
```javascript
// server/dialogue-memory.js
// 异步预计算嵌入向量（存储时调用，不阻塞主流程）
async function precomputeEmbedding(sessionId, round) {
  try {
    const text = round.userQuery + ' ' + (round.aiResponse || '');
    const embedding = await getEmbedding(text);
      
    // 存入内存缓存
    embeddingCache.set(round.roundId, embedding);
      
    // 存入磁盘缓存
    const cache = loadEmbeddingCache(sessionId);
    cache[round.roundId] = embedding;
    saveEmbeddingCache(sessionId, cache);
      
    console.log(`[DialogueMemory] 预计算嵌入向量成功: ${round.roundId}`);
  } catch (err) {
    console.warn(`[DialogueMemory] 预计算嵌入向量失败 ${round.roundId}:`, err.message);
  }
}

// 增强查询（指代消解 + 上下文补全）
async function enhanceQueryWithMemory(currentQuery, sessionId) {
  const { success, history, context } = getConversationHistory(sessionId, MAX_CONTEXT_LENGTH, true);
    
  if (!success || history.length === 0) {
    return { enhancedQuery: currentQuery, originalQuery: currentQuery, usedHistory: [], coreferences: {}, context: null };
  }
    
  // 1. 指代消解：替换指代词
  let enhancedQuery = currentQuery;
  const coreferences = context ? context.coreferences : {};
    
  for (const [pronoun, entity] of Object.entries(coreferences)) {
    if (enhancedQuery.includes(pronoun)) {
      enhancedQuery = enhancedQuery.replaceAll(pronoun, entity);
    }
  }
    
  // 2. 智能选择相关历史轮次（基于语义相似度）
  const relevantHistory = await selectRelevantHistory(currentQuery, enhancedQuery, history, sessionId);
    
  // 3. 构建上下文增强的查询（用于RAG检索）
  if (relevantHistory.length > 0) {
    const contextParts = [];
      
    for (const h of relevantHistory) {
      contextParts.push(`Q: ${h.userQuery}`;
      const answerSummary = h.aiResponse && h.aiResponse.length > 150 
        ? h.aiResponse.slice(0, 150) + '...' 
        : (h.aiResponse || '');
      if (answerSummary) {
        contextParts.push(`A: ${answerSummary}`);
      }
    }
      
    const contextPrefix = contextParts.join('\n');
    enhancedQuery = `【对话上下文】\n${contextPrefix}\n\n【当前问题】\n${enhancedQuery}`;
  }
    
  return {
    enhancedQuery,
    originalQuery: currentQuery,
    usedHistory: relevantHistory,
    coreferences,
    context: context ? { keyEntities: context.keyEntities, topicChain: context.topicChain.slice(-5) } : null
  };
}
```

// server/answer-rewriter.js
// 核心改写功能
async function rewriteToColloquial(originalAnswer, userMessage = '', options = {}) {
  const { tone = 'friendly', emotion = 'neutral' } = options;
    
  try {
    const prompt = buildRewritePrompt(originalAnswer, userMessage, { tone, emotion });
    const rewritten = await callOllamaGenerate(prompt, {
      baseURL: OLLAMA_BASE_URL,
      model: OLLAMA_MODEL,
      temperature: 0.7,
      max_tokens: 1000,
      timeout: 60000
    });
      
    return rewritten || originalAnswer;
  } catch (err) {
    console.error('[AnswerRewriter] 改写失败，使用降级方案:', err.message);
    // 降级方案：简单模板改写
    return `[${tone}语气] ${originalAnswer}`;
  }
}

// 质量评估
function evaluateQuality(original, rewritten) {
  // 多维度评估：流畅度、准确度、亲切度
  const fluency = calculateFluency(rewritten);
  const accuracy = calculateAccuracy(original, rewritten);
  const friendliness = calculateFriendliness(rewritten);
    
  const overallScore = (fluency + accuracy + friendliness) / 3;
    
  return {
    fluency,
    accuracy,
    friendliness,
    overallScore: Math.round(overallScore * 100) / 100
  };
}

## 🏗️ 技术架构分析

### 1. 技术选型亮点

#### ✅ 优点
1. **轻量级技术栈**：Node.js + Express + React，无需复杂配置，快速上手
2. **本地LLM部署**：Ollama部署qwen2.5:14b，数据不出网，安全性高
3. **JSON文件存储**：无需数据库，简单易维护，适合中小规模（<10万条FAQ）
4. **模块化设计**：核心AI模块独立封装，易于测试和替换

#### ⚠️ 改进方向
1. **JSON文件存储**：大规模数据（>10万条FAQ）时，读写性能下降，建议迁移到SQLite或MongoDB
2. **单机部署**：当前为单机模式，无法水平扩展，建议引入Redis + 负载均衡
3. **Ollama性能**：qwen2.5:14b响应较慢（2-5秒），建议升级到qwen2.5:7b或使用GPU加速
