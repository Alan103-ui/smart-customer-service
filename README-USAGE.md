# 广康AI智能客服系统 - 功能清单与使用指南

**版本**: v1.0.0  
**更新时间**: 2026-07-01  
**状态**: ✅ 基本完成，可投入使用

## 📋 功能清单

### 1. 用户端功能

#### 1.1 智能对话
- ✅ **多轮对话**: 支持上下文理解和指代消解
- ✅ **意图识别**: 自动识别用户意图（查询、投诉、建议等）
- ✅ **实体提取**: 提取关键信息（组织、产品、时间等）
- ✅ **对话记忆**: 跨会话记忆用户偏好和历史对话
- ✅ **答案改写**: 根据语调模板改写答案风格

#### 1.2 知识查询
- ✅ **FAQ查询**: 常见问题自动回答
- ✅ **知识库检索**: 基于RAG的智能检索
- ✅ **语义搜索**: 理解用户意图，返回最相关答案

---

### 2. 管理员端功能

#### 2.1 知识库管理
- ✅ **知识库管理**: 创建、编辑、删除知识库
- ✅ **分类管理**: 管理知识分类体系
- ✅ **FAQ管理**: 批量导入、编辑、删除FAQ
- ✅ **文档上传**: 支持多种格式（PDF、Word、TXT等）

#### 2.2 RAG管理
- ✅ **向量库统计**: 查看向量库状态（chunk数、模型信息等）
- ✅ **模型管理**: 查看可用嵌入模型（bge-m3、qwen2.5等）
- ✅ **重新向量化**: 支持选择不同模型重新向量化
- ✅ **RAG测试**: 测试RAG检索效果
- ✅ **RAG评估**: 自动评估RAG系统性能

#### 2.3 对话管理
- ✅ **对话历史**: 查看用户对话历史
- ✅ **对话记忆**: 查看和管理用户记忆
- ✅ **记忆统计**: 查看记忆使用情况

#### 2.4 用户管理
- ✅ **用户管理**: 添加、编辑、删除用户
- ✅ **权限管理**: 配置用户权限
- ✅ **组织架构**: 管理组织架构

#### 2.5 系统管理
- ✅ **系统统计**: 查看系统使用情况（FAQ数、知识库数等）
- ✅ **日志管理**: 查看系统日志
- ✅ **A8配置**: 配置致远A8+对接
- ✅ **SSO配置**: 配置单点登录

---

### 3. 技术特性

#### 3.1 RAG技术栈
- ✅ **嵌入模型**: bge-m3:latest（1024维）
- ✅ **重排序模型**: bge-reranker-v2-m3:latest
- ✅ **生成模型**: qwen2.5:14b
- ✅ **向量库**: 本地JSON存储（支持扩展为ChromaDB）
- ✅ **Rerank策略**: Python服务 → LLM → 软重排序（三重降级）

#### 3.2 认证与安全
- ✅ **JWT认证**: 基于Token的认证机制
- ✅ **权限控制**: 管理员/普通用户权限分离
- ✅ **SSO对接**: 支持致远A8+单点登录

#### 3.3 性能优化
- ✅ **向量缓存**: 嵌入向量缓存（内存+磁盘）
- ✅ **FAQ缓存**: 热门FAQ缓存
- ✅ **对话记忆缓存**: 内存缓存，提升读取速度

## 🚀 使用指南

### 1. 启动系统

#### 1.1 启动后端
```bash
cd server
node index.js
```
后端运行在 `http://localhost:3001`

#### 1.2 启动前端（开发模式）
```bash
npm run dev
```
前端运行在 `http://localhost:5173`

#### 1.3 启动Rerank服务（可选）
```bash
cd server
start-rerank-server.bat
```
Rerank服务运行在 `http://172.17.6.18:8000`

---

### 2. 登录系统

#### 2.1 管理员登录
- 用户名: `admin`
- 密码: `admin123`

#### 2.2 普通用户登录
- 需先在管理员后台创建用户

---

### 3. 配置知识库

#### 3.1 添加FAQ
1. 登录管理员后台
2. 进入"知识库管理" → "FAQ管理"
3. 点击"添加FAQ"，填写问题和答案
4. 保存

#### 3.2 上传文档
1. 进入"知识库管理" → "知识库列表"
2. 点击"上传文档"
3. 选择文件（支持PDF、Word、TXT等）
4. 系统自动向量化

---

### 4. 测试对话

#### 4.1 前端测试
1. 打开前端页面 `http://localhost:5173`
2. 在对话框输入问题
3. 查看回答和来源

#### 4.2 API测试
```bash
# 登录
curl -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 发送消息（需通过WebSocket）
```

---

### 5. 监控与优化

#### 5.1 查看系统统计
- 管理员后台 → 仪表盘 → 系统统计

#### 5.2 查看RAG性能
- 管理员后台 → RAG管理 → RAG评估

#### 5.3 优化建议
- 定期重新向量化（当知识库更新时）
- 调整RAG参数（topK、threshold等）
- 添加更多FAQ（提升命中率）
