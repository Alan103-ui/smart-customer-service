## 🔍 优化措施详解

### 1. 意图理解模块优化

#### 优化前的问题
1. **测试用例错误**：期望意图写错（`suggestion/feature` → 应为`suggestion/new_feature`）
2. **Prompt示例不足**：只有7个few-shot示例，LLM容易误分类
3. **无重试机制**：一次调用失败直接使用降级方案
4. **JSON解析容错不足**：简单正则匹配，容易解析失败

#### 优化措施
1. **修复测试用例错误**
   ```javascript
   // server/test-intent.js 第29行
   // 优化前
   { query: '建议增加微信支付', expectedIntent: 'suggestion/feature', ... }
   
   // 优化后
   { query: '建议增加微信支付', expectedIntent: 'suggestion/new_feature', ... }
   ```

2. **增强Prompt工程**
   ```
   Query: "建议增加微信支付"
   Output: {"primaryIntent":{"level1":"suggestion","level2":"new_feature","confidence":0.95},"entities":[{"type":"feature","value":"微信支付","confidence":0.9}]}
   
   Query: "客服电话是多少"
   Output: {"primaryIntent":{"level1":"query","level2":"contact","confidence":0.95},"entities":[{"type":"contact","value":"客服电话","confidence":0.95}]}
   
   Query: "系统太慢了"
   Output: {"primaryIntent":{"level1":"complaint","level2":"service","confidence":0.9},"entities":[]}
   ```

3. **实现重试机制**
   ```javascript
   // server/intent-understanding.js
   async function understandIntent(userQuery, context, retryCount = 0) {
     try {
       // ... LLM调用 ...
     } catch (err) {
       if (retryCount < 2) {
         await sleep(1000 * (retryCount + 1));  // 指数退避：1s, 2s, 3s
         return await understandIntent(userQuery, context, retryCount + 1);
       }
       return fallbackIntent(query);
     }
   }
   ```

4. **增强JSON解析容错**
   ```javascript
   // server/ollama-client.js
   async function callOllamaJSON(prompt, options = {}) {
     // 策略1：直接解析（最快）
     // 策略2：提取markdown代码块中的JSON
     // 策略3：正则匹配 + 智能修复
     // 策略4：降级为默认结构
   }
   
   function repairJSON(jsonStr) {
     // 修复1：移除可能的尾部逗号
     // 修复2：补全缺失的闭合括号
   }
   ```

### 2. 对话记忆模块优化

#### 优化前的问题
1. **关键词提取精度低**：简单按分隔符分割，无法处理中文
2. **无法理解语义相关性**：只能匹配完全相同的关键词
3. **无缓存机制**：每次查询都实时计算，响应慢
4. **无降级策略**：bge-m3失败时报错

#### 优化措施
1. **集成nodejieba中文分词库**
   ```javascript
   const jieba = require('nodejieba');
   
   function extractKeywords(text) {
     const words = jieba.cut(text, true);  // true = 精确模式
     // 过滤：长度>=2、不在停用词表中
   }
   ```

2. **集成bge-m3向量嵌入模型**
   ```javascript
   const { getEmbedding } = require('./bge-m3-client');
   
   async function selectRelevantHistory(originalQuery, enhancedQuery, history, sessionId = null) {
     const queryEmbedding = await getEmbedding(enhancedQuery);
     const similarity = cosineSimilarity(queryEmbedding, roundEmbedding);
   }
   ```

3. **实现三层缓存机制**
   ```javascript
   // 1. 内存缓存（最快）
   const embeddingCache = new Map();
   
   // 2. 磁盘缓存（持久化）
   function loadEmbeddingCache(sessionId) { ... }
   function saveEmbeddingCache(sessionId, cache) { ... }
   
   // 3. 实时计算（降级）
   if (!roundEmbedding) {
     roundEmbedding = await getEmbedding(roundText);
   }
   ```

4. **预计算嵌入向量**
   ```javascript
   async function precomputeEmbedding(sessionId, round) {
     const embedding = await getEmbedding(text);
     embeddingCache.set(round.roundId, embedding);  // 内存缓存
     cache[round.roundId] = embedding;  // 磁盘缓存
   }
   ```

### 3. 答案改写模块优化

#### 优化前的问题
1. **测试脚本参数格式错误**：`batchRewrite`期望`{ original }`对象数组，但测试脚本传递字符串数组
2. **质量评估缺少await**：`evaluateQuality`是异步函数，但测试脚本中没有`await`
3. **语气列表显示格式错误**：`getToneList`返回对象数组，但测试脚本中使用`.join(', ')`直接拼接

#### 优化措施
1. **修复批量改写参数格式**
   ```javascript
   // server/test-rewrite.js
   // 优化前
   const results = await batchRewrite(testCases.map(t => t.originalAnswer), { ... });
   
   // 优化后
   const items = testCases.map(t => ({ original: t.originalAnswer }));
   const results = await batchRewrite(items, { ... });
   ```

2. **修复质量评估缺少await**
   ```javascript
   // 优化前
   const quality = evaluateQuality(original, rewritten);
   
   // 优化后
   const quality = await evaluateQuality(original, rewritten);
   ```

3. **修复语气列表显示格式**

## ⚠️ 已知问题与解决

### 问题1：Ollama服务器偶发超时
- **现象**："报销政策是什么"导致Ollama超时（30秒）
- **原因**：Ollama服务器响应慢，默认超时不够
- **解决**：增加超时时间到90秒（`timeout: 90000`）
- **状态**：✅ 已解决

### 问题2：JSON解析失败
- **现象**：Ollama返回格式错误，JSON解析失败
- **原因**：Ollama偶尔返回不完整或格式错误的JSON
- **解决**：实现4层解析策略 + 智能JSON修复
- **状态**：✅ 已解决

### 问题3：嵌入向量缓存未持久化
- **现象**：重启后端服务器后，嵌入向量缓存丢失，首次查询仍然慢
- **原因**：只实现了内存缓存（Map），未持久化到磁盘
- **解决**：实现磁盘缓存（JSON文件），在`storeConversationRound`时预计算并存储
- **状态**：✅ 已解决

### 问题4：测试脚本路径格式错误
- **现象**：使用Windows路径格式（`D:\Clow\projects\...`）运行脚本，导致"文件未找到"
- **原因**：在Git Bash环境中使用了错误的路径格式
- **解决**：改为Git Bash路径格式（`/d/Clow/projects/...`）
- **状态**：✅ 已解决

### 问题5：bge-m3响应速度较慢
- **现象**：首次查询响应时间2-5秒
- **原因**：bge-m3模型计算量大
- **解决**：实现三层缓存机制，缓存命中响应时间<0.01秒
- **状态**：⚠️ 部分解决（首次查询仍慢）
